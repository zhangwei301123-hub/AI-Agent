# 想定态势数据推送结构设计文档 (`SendMsg`)

本文档描述了推演服务端推送至 Cesium 三维前端态势数据的主体结构 `SendMsg`，包括其字段、类型、嵌套子结构及相关枚举。主要用于指导前端开发以及接口态势数据对接。

---

# <a id="sendmsg">1. 核心推送结构: `SendMsg`</a>

`SendMsg` 类是推演服务端向前端推送实时态势及战损评估统计的主体数据结构。

| 字段名 | 类型 | 运行期实际含义与赋值逻辑 | `getSendMsg` 使用状态 |
| :--- | :--- | :--- | :--- |
| `Id` | `string` | **当前想定的文件物理路径**（例如：`D:\想定\xxx.scen`） | `✔️ 已使用` |
| `ScenName` | `string` | **想定标题/名称** | `✔️ 已使用` |
| `scenId` | `string` | **想定唯一标识 ID (GUID)** | `✔️ 已使用` |
| `msgCode` | [MsgCode](#msgcode) | **消息类型代码**。<br>• 常规态势推送时为 `三维获取态势` (`3`)；<br>• 战损/推演效能统计推送时为 `获取战损统计` (`6`) | `✔️ 已使用` |
| `UnitList` | `ConcurrentList<`[UnitInfo](#unitinfo)`>` | **当前活着的实体列表**。<br>• 非上帝视角下：包含己方存活实体及探测到的敌方/中立/未知“接触目标”；<br>• 上帝视角下：包含想定中全部存活的实体 | `✔️ 已使用` |
| `RectangleAreaPointList` | `ConcurrentList<List<`[Geopoint_Struct](#geopoint_struct)`>>` | **箔条干扰区顶点坐标列表**，存放当前场景中所有箔条云团的 4 顶点矩形坐标点集 | `✔️ 已使用` |
| `MapCenter_Longitude` | `double` | **视点地图中心点经度**。来自当前阵营主视角的经度（蒙特卡洛计算下硬编码为 `121`） | `✔️ 已使用` |
| `MapCenter_Latitude` | `double` | **视点地图中心点纬度**。来自当前阵营主视角的纬度（蒙特卡洛计算下硬编码为 `24`） | `✔️ 已使用` |
| `MapCenter_Altitude` | `double` | **视点相机高度 (米)**。来自当前阵营主视角的相机高度（蒙特卡洛计算下硬编码为 `50000`） | `✔️ 已使用` |
| `TimeCompression` | `int` | **推演倍速**，指示当前推演的运行速率。`0` 代表正常倍速，正值代表加速倍数 | `✔️ 已使用` |
| `SideGuid` | `string` | 当前消息所属阵营的 GUID。主要在订阅推送时用于路由分发与过滤 | `❌ 未使用` (仅订阅路由匹配用) |
| `SideColor` | `string` | 阵营颜色（保留字段，暂未直接赋值） | `❌ 未使用` (冗余声明) |
| `RemainderTime` | `string` | **推演剩余时间格式化文本**。格式为：`剩余:X天 X时 X分 X秒` | `✔️ 已使用` |
| `CurrentTime` | `string` | **想定当前推演的时刻（格式化本地时间）**，格式为 `yyyy-MM-dd HH:mm:ss` | `✔️ 已使用` |
| `SceneStatus` | `int` | **想定运行状态**。`0` 代表未开始（或暂停/停止），`1` 代表正在运行 | `✔️ 已使用` |
| `ScenXML` | `string` | 想定 XML 文件内容 | `❌ 未使用` (冗余声明) |
| `sideUnitDamageList` | `ConcurrentList<`[CommonModel](#commonmodel)`>` | **阵营累计受损实体总数列表**。在获取战损统计消息时有效，统计各阵营累计被毁的飞机、舰船、潜艇、设施及武器总和 | `⚠️ 仅战损接口使用` |
| `sideUnitDamageList2` | `ConcurrentList<`[CommonModel](#commonmodel)`>` | **对比组/第二数据源阵营实体损伤统计列表**。在对比历史战损时有效，用于读取历史对比值 | `⚠️ 仅战损接口使用` |
| `sideUnitConsumeList` | `ConcurrentList<`[CommonModel](#commonmodel)`>` | **阵营武器/弹药累计发射总数消耗列表**。在获取战损统计时有效 | `⚠️ 仅战损接口使用` |
| `sideUnitConsumeList2` | `ConcurrentList<`[CommonModel](#commonmodel)`>` | **对比组/第二数据源物资消耗/弹药发射对比列表**。在对比历史消耗时有效 | `⚠️ 仅战损接口使用` |
| `sideDetectionList` | `ConcurrentList<`[DetectionModel](#detectionmodel)`>` | **阵营各域目标探测数量统计列表**。在获取战损统计中有效，按序包含：陆地、空中、海面、水下、鱼雷、武器目标的探测总数 | `⚠️ 仅战损接口使用` |
| `sideWeaponList` | `ConcurrentList<`[sideWeaponModel](#sideweaponmodel)`>` | **阵营武器发射与效能统计列表**。在获取战损统计中有效，按序包含：发射数、命中数、脱靶数、故障数、拦截数、命中率 | `⚠️ 仅战损接口使用` |
| `sideWeaponList2` | `ConcurrentList<`[sideWeaponModel](#sideweaponmodel)`>` | **对比组/第二数据源武器发射与效能对比列表**。在对比历史武器效能时有效 | `⚠️ 仅战损接口使用` |
| `sideUnitList` | `ConcurrentList<`[sideUnitModel](#sideunitmodel)`>` | **阵营当前存活各类平台数量统计列表**。在获取战损统计中有效，按序包含：当前活着的飞机、潜艇、舰船、卫星、设施数量 | `⚠️ 仅战损接口使用` |
| `DeleteUnitIdList` | `ConcurrentList<`[UnitInfo](#unitinfo)`>` | **已被摧毁/删除的实体列表**，前端接收后执行 Cesium 模型与图标的移除 | `✔️ 已使用` |
| `AddedUnitList` | `ConcurrentList<`[UnitInfo](#unitinfo)`>` | 新增实体列表。已遗弃的字段，新增实体直接在 `UnitList` 中推送 | `❌ 未使用` (遗弃字段) |
| `radiationAndDataLinkLine` | [radiationAndDataLinkLine](#radiationanddatalinkline) | **空间物理/电磁连线态势数据**，包含数据链、雷达照射线、反辐射定位线、雷达/通信干扰压制线、激光武器线和武器引导线等 | `✔️ 已使用` |

# 2. 嵌套/关联子数据结构

### <a id="unitinfo">2.1 实体信息结构 `UnitInfo`</a>

`UnitInfo` 类代表想定中一个推演实体的当前状态和属性：

| 字段名 | 类型 | 运行期实际含义与赋值逻辑 | `getSendMsg` 使用状态 |
| :--- | :--- | :--- | :--- |
| `guid` | `string` | 实体唯一 GUID。若是编组，则是该编组的唯一 GUID | `✔️ 已使用` |
| `contactGuid` | `string` | 探测到的目标 GUID (仅在非上帝视角下，且探测到虚警/接触目标时有效) | `✔️ 已使用` |
| `name` | `string` | 实体呼号。若是编组则为编组名称；若是编组领队，会追加 `【领队】` 字样 | `✔️ 已使用` |
| `unitname` | `string` | 实体型号名称。若是编组，直接为编组名称 | `✔️ 已使用` |
| `longitude` | `double` | 实体实时经度（度） | `✔️ 已使用` |
| `latitude` | `double` | 实体实时纬度（度） | `✔️ 已使用` |
| `altitude` | `double` | 实体实时高度 (米) | `✔️ 已使用` |
| `heading` | `double` | 实体实时航向角 (度) | `✔️ 已使用` |
| `ColorStr` | `string` | 阵营颜色的 RGB 格式分量值（如 `"255,0,0"`） | `✔️ 已使用` |
| `side` | `string` | 所属阵营名称（如红方、蓝方等） | `✔️ 已使用` |
| `SideId` | `string` | 所属阵营 GUID | `✔️ 已使用` |
| `DBID` | `int` | 数据库对应的模板 ID，用于关联查询详细物理指标 | `✔️ 已使用` |
| `ComponentNumber` | `int` | 卫星平台关联的载荷组件数 | `✔️ 已使用` |
| `Icon2D` | `string` | 2D 军标/图标相对路径 | `✔️ 已使用` |
| `Icon3D` | `string` | 3D 模型在 Cesium 前端渲染加载对应的资产名称或路径 | `✔️ 已使用` |
| `unitModel` | `string` | 三维模型匹配标识。若在特定精细模型匹配列表中，则为具体型号名（如 `aim_9m` 等），否则为空 | `✔️ 已使用` |
| `unitType` | [ActiveUnitType](#activeunittype) | 实体子类型枚举（飞机/舰船/潜艇等） | `✔️ 已使用` |
| `unitCategory` | [UnitCategory](#unitcategory) | 实体大类枚举（如飞机、水面舰艇、武器等） | `✔️ 已使用` |
| `UnitSpecificType` | `int` | 提供给三维前端的平台小类编码，用于更细致的模型加载 | `✔️ 已使用` |
| `sideColor` | `string` | 阵营颜色的十六进制字符串，如 `"#FF0000"` | `✔️ 已使用` |
| `rangeDetect` | `double` | 探测范围半径。编组时为内部所有平台探测范围的最大值（公里） | `✔️ 已使用` |
| `rangeStrike_Air` | `double` | 对空打击最大射程（已自动换算为**公里/KM**） | `✔️ 已使用` |
| `rangeStrike_Surface` | `double` | 对海/水面打击最大射程（已自动换算为**公里/KM**） | `✔️ 已使用` |
| `rangeStrike_Land` | `double` | 对地打击最大射程（已自动换算为**公里/KM**） | `✔️ 已使用` |
| `rangeStrike_Submarine`| `double` | 对潜打击最大射程（已自动换算为**公里/KM**） | `✔️ 已使用` |
| `rangeUnguidedWeapon` | [WeaponMicrowave](#weaponmicrowave) | 无制导或微波武器的有效包络范围 | `✔️ 已使用` |
| `WeaponImpact` | `int` | 武器碰撞/打击触发状态 (默认为 `0`) | `✔️ 已使用` |
| `rangeSensor_Air` | `List<`[UnitAngle](#unitangle)`>` | 对空传感器的探测扇区范围列表 | `✔️ 已使用` |
| `rangeSensor_Sea` | `List<`[UnitAngle](#unitangle)`>` | 对海/对水面传感器的探测扇区范围列表 | `✔️ 已使用` |
| `rangeSensor_UnderWater` | `List<`[UnitAngle](#unitangle)`>` | 对水下传感器(声呐)的探测扇区范围列表 | `✔️ 已使用` |

| `rangeLaserWeapon` | `List<`[GeoPoint](#geopoint)`>` | 激光武器的照射线段点列表。此类上的该属性在运行期未被赋值，激光照射线通过全局态势线段汇总推送 | `❌ 未使用` (遗弃字段) |
| `RadarJammerResult_rangeSensor_Air` | `List<`[RadarJammerResult](#radarjammerresult)`>` | 对空雷达干扰计算得到的压制失效边界包络区域 | `✔️ 已使用` |
| `RadarJammerResult_rangeSensor_Sea` | `List<`[RadarJammerResult](#radarjammerresult)`>` | 对海雷达干扰计算得到的压制失效边界包络区域 | `✔️ 已使用` |
| `RadarJammerResult_rangeSensor_UnderWater` | `List<`[RadarJammerResult](#radarjammerresult)`>` | 对水下声呐干扰计算得到的压制失效边界包络区域 | `✔️ 已使用` |
| `CommunicationJammerResults` | `List<`[CommunicationJammerResult](#communicationjammerresult)`>` | 通信干扰算法计算所得的压制区域与畅通区域 | `✔️ 已使用` |
| `CommunicationJammerLinkResults` | [CommunicationJammerLinkResult](#communicationjammerlinkresult) | 通信断联或正常通信状态链路 | `✔️ 已使用` |
| `jammerBandAngles` | [JammerBandAngle](#jammerbandangle) | 干扰机的频段与有效干扰角度信息 | `✔️ 已使用` |

| `CurrentScenarioTime` | `DateTime` | 当前想定的推演时刻时间戳 | `✔️ 已使用` |
| `IsGroup` | `bool` | 该实体是否为编组 (如舰艇编队/飞行编队) | `✔️ 已使用` |
| `GroupType` | `string` | 编组类型的类别字符串名称 (如 `AirGroup` 或 `NavalGroup` 等) | `✔️ 已使用` |
| `Speed` | `double` | **实体实时速度**。<br>• 普通实体：单位为**公里每小时(KM/H) 或 节(Kts)**；<br>• 武器弹药 (IsWeapon == true)：自动转换为 **米每秒(m/s)** | `✔️ 已使用` |
| `BloodAmount` | `double` | **实体实时健康度/血量百分比** (0.0 ~ 100.0)。仅在通信状态正常时获取 | `✔️ 已使用` |
| `IsContact` | `bool` | 是否是正常推演视角中探测到的“接触目标” | `✔️ 已使用` |
| `ContactLastDetectTimeStr` | `string` | 目标最后一次被探测到距离当前时刻的格式化时间差文本 | `✔️ 已使用` |
| `WeaponGuidanceType_0`| `string` | 武器的主制导方式描述字符串（如果支持再次攻击，会追加 ` - 能够再次攻击` 提示） | `✔️ 已使用` |

| `CommText` | `string` | **通信中断状态字样**。若该实体未能接入通信网，显示为 `"通信中断"` | `✔️ 已使用` |
| `WeaponTargetDistance`| `double` | **武器距离所追踪目标的实时椭球体空间距离** (公里，保留两位小数) | `✔️ 已使用` |
| `WeaponTargetName` | `string` | 武器所追踪的目标呼号/名称 | `✔️ 已使用` |
| `IsWeapon` | `bool` | 当前实体是否为飞行/行驶中的武器弹药本身 | `✔️ 已使用` |
| `UnitAirLines` | `ConcurrentList<`[AirLinePoint](#airlinepoint)`>` | **实体的航线/计划路径点列表**。首点为实体当前位置，后续点为各导航点 | `✔️ 已使用` |
| `UncertainAreaList` | `ConcurrentList<`[Geopoint_Struct](#geopoint_struct)`>` | 接触目标的误差位置不确定区域点集列表 | `✔️ 已使用` |
| `sideId` | `string` | 阵营 GUID 冗余字段 | `✔️ 已使用` |

# 毁伤相关字段

关于想定推演系统推送与展示数据中，涉及“单位毁伤、被何种武器毁伤、武器追踪目标、以及武器打击最终结果”的相关字段及其运行期含义如下：

------

### 1. 实体自身毁伤与健康度字段

在实体信息 `UnitInfo` 及与其关联的 `UnitInfoStatus` 中，通过以下字段指示实体的健康与毁伤状态：

- `BloodAmount` (`double`)：
  - **含义**：实体的**实时健康度/血量百分比** (取值范围 `0.0` ~ `100.0`)。仅在通信状态正常时推送，实体血量为 `0` 时即被判定为摧毁。
- `Label_UnitDamage` (`string`)：
  - **含义**：**战损评估文本描述**（BDA, Battle Damage Assessment），如 `"BDA: 35%"`。用以直接展示该单位受到的毁伤百分比。
- `PB_FireDamage` / `Label_FireDamage` (`double` / `string`)：
  - **含义**：**火灾毁伤强度百分比**（0, 25, 50, 75, 100）。指征当前实体内部发生火灾的严重程度。
- `PB_FloodDamage` / `Label_FloodDamage` (`double` / `string`)：
  - **含义**：**浸水毁伤强度百分比**（0, 25, 50, 75, 100）。常用于水面舰船和潜艇遭遇鱼雷/水雷等攻击后的船体破损进水评估。
- 各载荷组件（Mount、Magazine、Sensor、CommDevice）的状态字段：
  - 含义：在损毁控制详情中，每个具体挂载、弹药库、传感器或通信/数据链设备均有其工作状态（PlatformComponent._ComponentStatus）：
    - `正常工作`
    - `有毁伤`（细分为轻度 `Light`、中度 `Medium`、重度 `Heavy`）
    - `已损坏`

------

### 2. 武器的目标与追踪距离字段

当实体本身作为正在飞行的武器弹药（`IsWeapon == true`）时，通过以下字段指示其目标和状态：

- `WeaponTargetName` (`string`)：
  - **含义**：当前武器**正在追踪锁定的目标单位呼号/名称**。
- `WeaponTargetDistance` (`double`)：
  - **含义**：当前武器距离所追踪目标实体的**实时空间椭球体距离**（单位：**公里/KM**）。

------

### 3. 武器碰撞与打击触发字段

- `WeaponImpact` (`int`)：
  - **含义**：当前实体是否正发生**武器碰撞/打击引爆触发状态**（在运行期通过 `GetWeaponImpact(ActiveUnit)` 动态获取最近一次碰撞的类型）。
  - 取值定义：
    - `0`：正常（未受到武器引爆打击）。
    - `1`：**电磁/电子压制碰撞**（`Electronic`，如受到高功率微波武器或严重电磁压制，前端一般挂载 `Impact_Electronic.png` 爆炸效果）。
    - `2`：**动能/物理爆炸击中**（`Kinetic`，如受到导弹、鱼雷或常规弹药物理击中，前端挂载 `Impact.png` 爆炸效果）。
- `DeleteUnitIdList` (`ConcurrentList<UnitInfo>`)：
  - **含义**：**当前周期已被摧毁并删除的实体列表**，前端收到后会在 Cesium 三维场景中移除该模型及图标。

------

### 4. 实时推演事件日志消息（`LoggedMessage`）

后台推演引擎在发生战斗结果（如击中、坠毁、脱靶等）时，会将事件输出为不同类型的事件日志（`LoggedMessage.MessageType`），主要通过前端日志面板展示：

- `LoggedMessage.MessageType.UnitLost`(单位损失) /`UnitDamage`(单位受损)：

  - **含义**：当某平台彻底被毁或受损时，生成诸如 `"[红方] F-16 战斗机 已被 [蓝方] 击毁"` 或是受创百分比的文本报告。

- `LoggedMessage.MessageType.WeaponEndgame`(武器最终结局)：

  - 含义

    ：详细记录发射出去的导弹/武器在终点阶段的物理结果，常见日志输出格式包括：

    - **物理击中/起爆** (Impact)：`"Weapon: AIM-120D has impacted F-16."` (武器击中了目标单位，伴有穿透、起爆或近炸引信触发等文本)。
    - **撞地/坠毁** (Smashed/Crashed)：`"Weapon: [导弹名称] has smashed into the ground."` (武器偏离航道或失控坠毁/撞地)。
    - **脱靶** (Overshot/Missed)：`"Weapon: [导弹名称] overshot its target."` (导弹飞越目标但未成功引爆/脱靶)。
    - **目标提前被毁** (Had target destroyed)：`"Weapon: [导弹名称] had its target destroyed..."` (所追踪的目标已提前被其他武器消灭，导弹失去目标)。
    - **失去引导信号/致盲** (Cannot have its target illuminated)：`"Weapon: [导弹名称] cannot have its target illuminated..."` (照射雷达关机/被毁，导致武器失去末导信号而盲飞)。

------

### 5. 战损与效能累计统计字段

在 `SendMsg` 的战损统计报表信息（`msgCode == 6`）中：

- **`sideUnitDamageList`**：统计各阵营当前**累计被摧毁的飞机、舰船、潜艇、设施及武器**的总和。

- `sideWeaponList`的`sideWeaponData` 数组：

  **含义**：对应阵营武器效能的累积统计，数组内 6 个槽位的值依次代表： `[ 武器发射总数, 武器命中总数, 武器脱靶总数, 武器故障总数, 武器被拦截数, 综合命中率 ]`
